Repo erstellen
  1. Wähle in der rechten oberen Ecke einer beliebigen Seite , und klicke dann auf Neues Repository.
  2. Gib einen kurzen, einprägsamen Namen für das Repository ein und wähle den "Owner" aus
  3. Wähle eine Sichtparkeitsoption aus
  4. README und Respository erstellen
Git Befehle Basic
  • git add . -> fügt eine Änderung aus dem Arbeitsverzeichnis zur Staging-area hinzu.
  • git add FILENAME -> hiermit kann nur eine Datei in die Staging-area hinzufügen.
  • git commit -m "Text was geändert wurde" -> erstellt einen neunen commit mit einer Nachricht "Text was geändert wurde"
  • git push -> Uploaded die Dateien die zuvor mit " git add . " in die Stagging area bewegt worden sind in das Github Repo
  • git log -> alle erstellten Commits werden im Terminal angezeigt (mit "q" kann ich die Ansicht verlassen)
  • git status -> kann verwendet werden, um zusammenzufassen, welche Dateien geändert und die für den nächsten Commit bereitgestellt wurden.
  • git pull -> mit "git pull" lade ich die geänderten Datein herunter
Branch

Verwende einen Branch, um die Entwicklungsarbeit ohne Auswirkungen auf andere Branches im Repository zu isolieren. Jedes Repository hat einen Standardbranch (main) und kann mehrere weitere Branches haben.

  • git branch -> zeigt alle vorhanden Branches an. (* ist wo man sich gerade befindet)
  • git branch BurgerMenu -> legt einen neunen Branch an, mit dem Namen BurgerMenu
  • git checkout BurgerMenu -> wechselt zu einem neuen Branch
  • git checkout -b BurgerMenu2 -> legft einen neunen Branch BurgerMenu2 an und wechselt in den neuen Branch (shortcut git branch und git checkout)
  • git branch -d BurgerMenu2 -> Löscht den Branch BurgerMenu2
Merge

Mit git merge werden mehrere Commit-Abfolgen in einen einheitlichen Verlauf zusammengeführt. Vor allem wird git merge genutzt, um zwei Branches zu vereinen.

  1. git checkout main -> Wechselt zum main Branch, was die Voraussetzung für einen Merge ist
  2. git merge BurgerMenu -> Führt den main Branch mit den BurgerMenu Branch (Beispiel) zusammen
  3. Fertigstellen mit git add. -> git commit -m "merge branch" -> git push
  4. git branch -d BurgerMenu -> Löscht den branch
Commit zurück gehen

Einen Commit zurück gehen ist manchmal notwendig und Sinnvoll. Das kann der Fall sein, wenn Fehler auftreten und es einfacher ist einen oder mehrere Schritte zurück zugehen

  1. git log --oneline -> ID des commits raussuchen zudem man zurück will
  2. git checkout 3290492 . -> Durch die ID zum alten Stand zurück
  3. Fertigstellen mit git add. -> git commit -m "back" -> git push
